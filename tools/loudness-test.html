为什么用粉噪? 粉噪频谱接近自然声音(音乐/语音的近似),响度稳定没有瞬时峰值波动,LUFS 几秒内就稳定,适合做测试基准。 比"随便找个视频"靠谱得多——那个视频响度可能恰好接近 -14,所以调整只有 ±2dB,根本听不出来。

② 一键预设

数字是测试页的输出增益(dBFS)。粉噪的实际 LUFS 会比这个值再低约 10 dB(粉噪 RMS 低于峰值)。

③ 预期行为(target = -14 时)

操作:扩展 popup target 设 -14 → 点播放 → 等 3~5 秒(LUFS 累积) → 看 popup 里这个 tab 的 gain 数值。

测试页增益近似 LUFS预期 gain听感
-45~-55+12 (触上限)强烈变响
-30~-40+12 (触上限)明显变响
-20~-30+12 (触上限)明显变响
-12~-22+8变响
-4~-14~0 (已平衡)几乎不变

再试试拖 popup 的 target:
• 拖到 -30 → 所有 gain 变负(衰减,变小)
• 拖到 -5 → 极轻的也提升到 +12(变响)

④ 多 tab 平衡(扩展的核心价值)

单个 tab 只验证"增益调整"。真正的价值是多个 tab 互相平衡:

  1. 开两个标签页都打开本页(复制 URL 到新标签)
  2. Tab A 设 -45(极轻),Tab B 设 -4(很响)
  3. 都点播放,扩展 target 保持 -14
  4. 等 5 秒——Tab A 的 gain 应到 +12,Tab B 应到 ~0
  5. 两个 tab 听起来响度趋近,这就是"平衡"
排查:如果扩展图标没显示绿色数字,说明这个页没被捕获(可能 Chrome 没把它标成 audible)。把测试页增益调到 -4 大声播放、或刷新扩展重试。Badge 有数字是一切的前提。
